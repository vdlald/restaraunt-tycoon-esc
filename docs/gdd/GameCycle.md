# Game Design Document

## 1. Общая концепция
- **Жанр:** Rogue-like Tycoon Restaurant / симулятор ресторана 
- **Цель:** Пройти 20 дней, корректно расставляя фурнитуру и обслуживая гостей

---

## 2. Игровой цикл (State Machine)

### 2.1. Лобби (Lobby)
| Параметр | Значение |
|----------|----------|
| Вход | Запуск игры / поражение / победа на 20-м дне |
| Карта | Пустая (нет фурнитуры, гостей, интерактивных объектов) |
| Взаимодействия | Запрещены |
| Настройки | Нет |
| Переход | Нажатие **P** → обратный отсчёт **3 сек** → «Подготовка» |
| Отмена | Повторное нажатие **P** во время отсчёта сбрасывает таймер |

### 2.2. Game / Подготовка (Preparation)
| Параметр | Значение |
|----------|----------|
| Вход | Из лобби или после успешного дня (мгновенный переход) |
| Доступные действия | Создание и расстановка фурнитуры из **начального набора** |
| Сохранение расстановки | Да, переносится между днями |
| Переход | Нажатие **P** → обратный отсчёт **3 сек** → «Обслуживание» |
| Отмена | Повторное нажатие **P** во время отсчёта сбрасывает таймер |

### 2.3. Game / Обслуживание (Service)
| Параметр | Значение |
|----------|----------|
| Вход | Из подготовки по таймеру |
| Доступные действия | Кормление гостей с помощью **Dish** |
| Запрещённые действия | Перемещение фурнитуры |
| Появление гостей | Все разом в начале фазы |
| Исчезновение гостей | После получения Dish |
| Появляющиеся элементы | Счётчик оставшихся гостей, таймер обратного отсчёта |
| Таймер | Зависит от количества гостей |
| Условие победы | Все гости накормлены → мгновенный переход в «Подготовку», **Day + 1** |
| Условие поражения | Таймер закончился, остался хотя бы 1 голодный гость → «Лобби» |
| Финал | День 20 → победный экран → «Лобби» |

---

## 3. Прогрессия

Выделить функцию, которая на входе принимает день, а на выходе количество гостей и таймер

---

## 4. Сущности

### 4.1. Игрок
- Собирает Dish (уже реализовано)
- Расставляет фурнитуру в подготовке
- Кормит гостей в обслуживании

### 4.2. Фурнитура
- **Источник:** начальный набор (ограниченный)
- **Доступность перемещения:** только в режиме подготовки
- **Сохранение:** конфигурация переносится между днями

### 4.3. Гости (NPC)
- Появляются все одновременно при старте обслуживания
- Требуют Dish
- Исчезают после кормления
- Счётчик отображает количество оставшихся

### 4.4. Dish
- Игрок собирает сам (уже реализовано)
- Используется для обслуживания гостей
